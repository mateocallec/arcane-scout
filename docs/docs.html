Getting Started

Arcane Scout is a Chrome / Firefox DevTools extension. It injects two panels directly into the browser's built-in developer tools:

  • API Inspector — captures and analyses every XHR and Fetch request made by the inspected page.
  • Pentest Tools — an integrated toolkit for manual web-application security testing.

Opening the extension

1
Press F12 (or Cmd⌘ Opt⌥ I on Mac) to open DevTools on any page.
2
Find the Arcane Scout tab in the DevTools tab bar at the top. It may be hidden behind the » overflow menu if the window is narrow.
3
The extension opens on the API Inspector panel. Use the tabs to switch between Requests, Routes, Header Auditor, Cookies, and Pentest Tools.
This page can be re-opened at any time via the extension icon in the toolbar or the right-click context menu "Arcane Scout — Help & Docs".

API Inspector

Requests are captured automatically the moment the panel is open. Reload the page or trigger interactions to populate the table. Only XHR and Fetch traffic is captured — navigation, service workers, and static assets are not shown.

Requests tab

The main request table. Each row shows:

ColumnDescription
MethodHTTP verb, colour-coded: GET blue, POST green, PUT amber, DELETE red, PATCH purple.
EndpointHost + path + query string. Hover to see the full URL.
StatusHTTP response code. Green for 2xx, amber for 3xx/4xx, red for 5xx.
TimeWall-clock timestamp of the request.

Use the filter bar to narrow rows by URL or method. Click any row to open the Detail Drawer.

The Export ▾ button saves captured traffic as JSON or HAR. Clear removes all captured requests from the current session.

Routes tab

A collapsible tree of all captured path segments. Click a node to filter the Requests tab to that route prefix. Click ✕ in the filter bar to clear the filter.

Header Auditor

Analyses every captured response for missing or misconfigured security headers, including:

  • Content-Security-Policy
  • Strict-Transport-Security
  • X-Frame-Options
  • X-Content-Type-Options
  • Referrer-Policy
  • Permissions-Policy

Each entry shows the header name, its current value (or absence), and a severity rating.

Cookies tab

Lists all cookies set on the current page's domain. Each row has a checkbox — uncheck it to delete the cookie from the browser for the current session. Re-check to restore it.

Cookie disable state persists across page reloads but is cleared when the tab is closed. Click Refresh to reload the list after a page interaction.

Detail Drawer

Click any request row to open the detail drawer. It has five tabs:

TabContents
OverviewURL, method, status, timing, response size, MIME type.
HeadersRequest and response headers in a searchable table.
PayloadRequest body — JSON pretty-printed, form data decoded, raw fallback.
ResponseResponse body — JSON pretty-printed, HTML sandboxed preview, image / video / audio viewers, raw text fallback.
ReplayPre-fills the Custom Request tab with this request's data so you can modify and resend it.

The cURL button in the Overview tab copies a ready-to-run curl command to the clipboard, including all request headers, body, and the -L --compressed flags.

Right-clicking any row in the HTTP Downgrade or HTTP Inspector result tables shows a "Send to Custom Request" context menu item.

Pentest Tools

Access all tools via the Pentest Tools tab inside DevTools. The tabs across the top of the panel switch between tools.

Encoder / Decoder

Converts data between arbitrary encoding schemes in a single click. Supported formats:

Plain Text (UTF-8) Base64 Base64 URL Hex Binary (01…) URL Encoded Double URL Encoded HTML Entities

Select the From and To formats, paste input, and click Convert. Use ⇅ Swap to reverse the conversion.

JWT Analyzer

Paste a JSON Web Token to instantly decode its header and payload. Features:

  • Header and payload displayed as syntax-highlighted JSON.
  • Standard claims table (iss, sub, exp, iat, etc.) with human-readable timestamps.
  • EXPIRED / VALID badge based on the exp claim.
  • Re-encode — edit the payload JSON and re-sign with alg: none for testing broken JWT verification.
The browser cannot verify a JWT signature — it only decodes. Re-encoding uses alg: none (no signature). Use this only in controlled test environments.

Payload Generator

Generates ready-to-use attack payloads. Three categories:

CategoryContexts / Options
XSS HTML tag, Attribute, JavaScript, SVG, Filter bypass. Or select All contexts for everything at once.
SQL Injection MySQL / MariaDB, PostgreSQL, MSSQL, SQLite, Oracle. Covers UNION, stacked, time-based blind, OOB, and privilege escalation.
CSRF PoC Generates a self-submitting HTML form targeting a URL. Add body parameters (one per line in name=value format).

Click the ⎘ button next to any payload to copy it individually. Copy All copies every payload in the current list.

Custom Request

A full HTTP request builder. Set any method, URL, headers, cookies, query parameters, and body. Requests run from the extension context — no CORS restrictions apply.

Supported body types: None, Raw, JSON (validated before sending), Form URL-encoded, XML.

The response area shows status, elapsed time, body size, and response headers. The body viewer auto-detects HTML (sandboxed iframe), images, video, audio, and JSON.

The Replay tab in the Detail Drawer pre-fills this tool with any captured request's data.

JSON Tools

Paste any JSON and click Pretty Print to format it with syntax highlighting, or Minify to compact it. Errors are reported with the failing position.

XOR

XOR a value against a key, both with configurable encoding. Input can be plain text, hex, or Base64. The key can be plain text or hex. Output can be hex, plain text, or Base64. Useful for obfuscated tokens and simple cipher analysis.

HTTP Downgrade

Tests whether the resources loaded by the current page are accessible over plain HTTP (i.e., vulnerable to SSLstrip / MitM interception).

How it works: clicking Start Test reloads the page, captures every network resource the browser requests, then concurrently probes each HTTPS URL with both an HTTP and an HTTPS HEAD request.

Each resource is classified into one of six categories:

ResultMeaning
VULNERABLEHTTP responds with 2xx — plaintext is fully accessible.
REDIRECT / NO HSTSHTTP redirects to HTTPS, but the HTTPS response has no Strict-Transport-Security header — the redirect can be intercepted.
REDIRECT + HSTSHTTP redirects to HTTPS, and HTTPS sends an HSTS header.
HSTSThe browser blocked the HTTP request internally (HSTS preload or cached policy).
HTTPS ONLYPort 80 is unreachable — the server only listens on HTTPS.
UNREACHABLEBoth HTTP and HTTPS probes failed (timeout, DNS failure, etc.).
This tool triggers a full page reload. Any unsaved state in the inspected page will be lost.

HTTP Inspector

Reloads the page and captures every plain HTTP request the browser makes: scripts, XHR/fetch, images, fonts, CSS, form submissions, iframes, and more.

Any resource loaded over HTTP is a potential mixed-content or interception risk. Click a row to open the Detail Drawer with full headers and response body.

Like HTTP Downgrade, this tool triggers a full page reload.

Vuln Disclosure

Fetches /.well-known/security.txt (RFC 9116) from the current domain and common variants (www. prefix, apex domain). Opens automatically when the tab is first selected.

Parsed fields:

Contact Expires Encryption Acknowledgments Preferred-Languages Canonical Policy Hiring

URL fields are rendered as clickable links. The Expires field shows a VALID / EXPIRED badge. Click Open Raw ↗ to view the raw file in a new tab.
